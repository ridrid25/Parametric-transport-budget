# Параметрическая модель транспортного бюджета

Модель отвечает на главный вопрос: **как меняются транспортные расходы при изменении выручки** — и, шире, при изменении числа клиентов, цены, стоимости топлива, ставки аутсорса и размера собственного парка.

Модель «параметрическая» в строгом смысле: расходы не введены цифрами, а **вычисляются как формулы от именованных параметров**. Меняешь параметр — вся цепочка `выручка → объём → рейсы → км → топливо/ТО/платные дороги/суточные/аутсорс` пересчитывается сама.

В комплекте два способа работы:

1. **`apps-script/Code.gs`** — скрипт для Google Sheets (Apps Script). Сам генерирует релевантные тестовые данные, строит бюджет, сценарный анализ и дашборд из 14 графиков прямо в таблице.
2. **`dashboard/dashboard.html`** — автономный интерактивный дашборд (один HTML-файл, без сервера и внешних зависимостей). Двигаешь ползунки — графики пересчитываются вживую. Работает на десктопе и телефоне, поддерживает светлую/тёмную тему, можно поделиться параметрами по ссылке.

## 1. Google Sheets (Apps Script)

### Установка

1. Создайте новую Google-таблицу.
2. **Расширения → Apps Script**.
3. Удалите содержимое `Code.gs` в редакторе и вставьте туда всё из [`apps-script/Code.gs`](apps-script/Code.gs).
4. Сохраните (💾) и обновите вкладку с таблицей.
5. В меню появится пункт **«Транспортный бюджет»**.

### Использование

Меню **«Транспортный бюджет»**:

| Пункт | Что делает |
|---|---|
| **Собрать всё с нуля** | Генерирует данные + бюджет + сценарии + разбивки + дашборд одной командой |
| 0. Сгенерировать тестовые данные | 13 машин, ~1000 клиентов (5–2000 км), сезонность, лист параметров |
| 1. Пересчитать бюджет | Годовой бюджет по месяцам и статьям затрат |
| 2. Сценарный анализ выручки | Как меняются расходы при ±объёме выручки (форма эластичности) |
| 3. Разбивка | По видам транспорта, регионам, топ-клиентам, дистанции, сегментам |
| 4. Построить дашборд | 14 графиков на отдельном листе |

Начните с **«Собрать всё с нуля»**, дальше меняйте значения на листе **`Params`** и жмите нужные пункты пересчёта.

### Параметры (лист `Params`)

| Параметр | По умолч. | Смысл |
|---|---|---|
| `fuel_price_rub_per_l` | 62 | Цена топлива, руб/л |
| `perdiem_rate_rub_per_night` | 2500 | Суточные за ночёвку в рейсе |
| `toll_free_km` | 50 | Первые N км без платных дорог |
| `toll_rate_rub_per_km` | 1.2 | Тариф платных дорог сверх лимита |
| `outsource_rate_rub_per_km` | 45 | Базовая ставка 3PL (1-й тариф) |
| `outsource_tier2_threshold_km_month` | 200000 | Порог км/мес для скидочного 2-го тарифа 3PL |
| `outsource_tier2_rate_rub_per_km` | 42 | Ставка 3PL выше порога 2 |
| `outsource_tier3_threshold_km_month` | 350000 | Порог км/мес для 3-го тарифа 3PL |
| `outsource_tier3_rate_rub_per_km` | 38 | Ставка 3PL выше порога 3 |
| `work_days_per_month` | 22 | Рабочих дней в месяце |
| `load_utilization` | 0.85 | Коэффициент использования грузоподъёмности |
| `overhead_monthly_fixed_rub` | 350000 | Постоянные накладные логистики |
| `revenue_scenario_pct` | 0 | Рост/падение объёма у **существующих** клиентов, % — расходы растут пропорционально |
| `customer_count_growth_pct` | 0 | Рост/отток **числа** клиентов, % — реально добавляет/убирает рейсы и км |
| `price_growth_pct` | 0 | Рост цены за кг, % — выручка растёт, пробег и расходы нет (чистая маржа) |
| `additional_vehicles_light` | 0 | Докупка лёгких машин сверх текущих 5 |
| `additional_vehicles_medium` | 0 | Докупка средних машин сверх текущих 5 |
| `additional_vehicles_heavy` | 0 | Докупка тяжёлых машин (фур) сверх текущих 3 |

### Что заложено в модель

- **Три тира транспорта:** лёгкий (Газели, до 3 т), средний (5–10 т), тяжёлый (фуры, 15–20 т). Собственный парк — 13 машин.
- **Драйверная себестоимость:** топливо (с сезонным индексом цены), ТО, платные дороги, оплата водителя, суточные — всё как функция км и рейсов.
- **Эффект масштаба / операционный рычаг:** постоянные расходы размываются с ростом объёма, но при исчерпании мощности парка добавляется маржинальный аутсорс.
- **Ступенчатая ставка 3PL** от суммарного аутсорс-пробега по всей компании (скидка за объём).
- **CAPEX vs аутсорс:** лист сравнивает докупку машин против аутсорса с учётом того, что тариф 3PL общий на компанию — докупка в одном тире может сдвинуть скидочную ступень для остальных.
- **Реалистичная нагрузка:** 13 машин на ~1000 клиентов дают высокую долю аутсорса (77–82%) — это особенность модели, а не ошибка, и она явно задокументирована в комментариях скрипта.

## 2. Интерактивный дашборд (`dashboard/dashboard.html`)

Полностью автономный HTML-файл: логика расчёта из `Code.gs` портирована на клиентский JS, никакого бэкенда. Откройте файл в браузере — или опубликуйте как Artifact.

Возможности:

- **Живой пересчёт** при движении ползунков (объём, число клиентов, цена, топливо, ставка аутсорса, докупка машин).
- **14 графиков:** бюджет по месяцам, структура постоянные/переменные, статьи затрат, эластичность, виды транспорта, загрузка парка, регионы, топ-клиенты, сегменты, дистанция, себестоимость км, CAPEX vs аутсорс.
- **Поделиться параметрами:** состояние сохраняется в URL-хэш и `localStorage` — ссылку можно скинуть коллеге, откроется ровно с вашими значениями.
- **Выгрузка в CSV:** кнопка «⬇ CSV» скачивает помесячный бюджет текущего сценария (выручка, все статьи затрат, итог, доля в выручке) с шапкой выбранных параметров — открывается в Excel/Google Sheets.
- **Мобильная версия:** сворачиваемая панель параметров, корректная прокрутка на iOS/Android.
- **Переключатель «Компьютер / Телефон»** вверху страницы — показывает дашборд в узкой мобильной колонке или во всю ширину, чтобы сразу видеть оба вида (работает в любом браузере, не только в панели артефактов).
- **Светлая/тёмная тема** подхватывается автоматически.

## Структура репозитория

```
apps-script/Code.gs        — скрипт для Google Sheets (генерация данных + бюджет + дашборд)
dashboard/dashboard.html   — автономный интерактивный дашборд
samples/                   — готовые образцы данных (xlsx + csv для Google Sheets)
tools/                     — скрипты проверки модели и сборки образцов (без Google Sheets)
tools/legacy/              — ранние версии (исторический генератор и статичный предпросмотр)
README.md
```

Подробности по инструментам — в [`tools/README.md`](tools/README.md): `harness.js` прогоняет `Code.gs` через заглушку Apps Script и проверяет сходимость расчётов, `export_inputs.js` + `build_sample_data.py` пересобирают образцы в `samples/`.
